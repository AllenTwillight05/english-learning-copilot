9. Algorithm View — FSRS-6 Spaced Repetition Scheduler

9.1 Overview

词汇复习模块采用 FSRS-6（Free Spaced Repetition Scheduler v6）算法，取代传统的固定间隔复习策略。FSRS 是一种基于机器学习的现代间隔重复算法，已由 open-spaced-repetition 社区发布为开放标准，最新版本为 v6，使用 21 个参数控制记忆状态的动态演化。

核心思想：每道题在用户复习后，系统根据用户的自我评分（1-4 四个等级）和该题的当前记忆状态，计算该题的新难度（Difficulty）、新稳定性（Stability）和下一次复习时间。答得好的题间隔变长，答得差的题间隔缩短——每个人的复习节奏完全个性化。

选型理由：相较于 Anki 的 SM-2 算法，FSRS-6 的优势在于：① 21 参数模型比 SM-2 的 6 参数更精准；② 内置遗忘曲线模型，可根据实际记忆留存率动态调整间隔；③ 是 DSR 模型（Difficulty-Stability-Retrievability）的工程化实现，有扎实的认知科学基础。

9.3 Algorithm Flow

9.4 Key Formulas

9.4.1 遗忘曲线 Forgetting Curve

R(t, S) = (1 + factor × t / S)^(-w₂₀)
其中 factor = 0.9^(-1/w₂₀) - 1，保证 t=S 时 R=90%（目标留存率）

给定两次复习间经过的天数 t 和当前稳定性 S，计算记忆留存率 R。R 越接近 0 表示遗忘越彻底。

9.4.2 反向间隔计算

I(r, S) = S / factor × (r^(1/(-w₂₀)) - 1)

给定目标留存率 r（默认 0.9）和稳定性 S，计算下次复习应隔多少天。结果经 ±5% 随机抖动（fuzz），防止批量题目总在一天到期。

9.4.3 首次初始化

初始稳定性： S₀ = w[rating-1]
Again=0.212, Hard=1.293, Good=2.307, Easy=8.296

初始难度： D₀ = w₄ - e^(w₅ × (G-1)) + 1
Again=6.41, Hard=4.82, Good=3.27, Easy=1.67

9.4.4 难度更新

ΔD = -w₆ × (G - 3)
Step 1: D₁ = D + ΔD × (10 - D) / 9
Step 2: D₂ = w₇ × D₀(Easy) + (1-w₇) × D₁
最终 clamp 到 [1, 10]

G=3(Good) 时 ΔD=0，难度不变。w₆=3.02 控制评分对难度的冲击。w₇=0.001 控制均值回归力度（几乎不做回归）。

9.4.5 稳定性更新

场景公式直觉
跨天答对
(recall)		 S' = S × (1 + e^w₈ × (11-D) × S^(-w₉) × (e^(w₁₀×(1-R))-1) × penalty × bonus) D↑→涨得少; S↑→涨得少; R↓(隔得久但没忘)→大涨（间隔效应）
同日复习
(short-term)		 S' = S × e^(w₁₇×(G-3+w₁₈)) × S^(-w₁₉) 同一学习会话内的短期强化
答错遗忘
(forget)		 S' = w₁₁ × D^(-w₁₂) × ((S+1)^w₁₃-1) × e^(w₁₄×(1-R))
且 S' ≤ S / e^(w₁₇×w₁₈)		 遗忘后重新估计稳定性，含补偿机制

9.5 System Integration

Data Flow: ReviewService 维护一个 FSRS 单例（requestRetention=0.9, defaultParams）。每次用户提交评分 → Controller 获取 username + questionId + rating → Service 查 user_word_progress 旧状态 → 映射为 CardState → fsrs.review(card, rating) → 新 CardState 写回 user_word_progress。获取到期词汇时，Service 查询 WHERE user_id=? AND due <= NOW()，关联 vocabulary 表返回完整题目。
