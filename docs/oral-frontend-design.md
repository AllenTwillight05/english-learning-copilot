## 流程说明

1. **入口**：从 `Home` 进入 `口语模块`。
2. **准备阶段**：
   - 进入 `口语界面`，展示多个情景模块。
   - 选择 `情景模块` 并阅读 `情景细节`（包含练习目标、难度、时长、关键词）。
   - 点击 `开始按钮` 进入练习。
3. **练习阶段（会话界面）**：
   - 提供三种设计风格供选择或参考：
     - **设计①**：以微信对话聊天框模式展现。（目前选择）
     - **设计②**：背景使用场景特定素材，对话仍以气泡形式出现（设计①的扩展版）。
     - **设计③**：模拟视频通话界面。
   - 若想结束联系，可点击 `交卷按钮`（至少对话3轮后方可交卷）。
4. **结果阶段**：
   - 进入 `评分界面`，展示反馈信息
        - 总分
        - 发音准确性
        - 流畅度
        - 语速
        - 问题句子合集
   - 点击 `退出` 结束当前流程。重新进入 `口语界面` 开始新的练习循环。
   - 或点击 `再练一次` 回到会话界面，重做当前练习。
5. **对应状态**
    - 口语界面：scenarios
    - 情景细节：preview
    - 会话界面：ready, recording, uploading
    - 交卷：evaluating
    - 评分界面：feedback
    - 错误：error
6. **技术栈**
    - 前端：
        - React, Ant Design
        - Wavesurfer.js: 界面上绘制用户录音的声波图
        - Recorder.js/MediaRecorder API: 浏览器原生API，捕获麦克风音频流，转换为Blob对象发送给后端
    - 后端：
        - java(暂定), Spring Boot, My SQL
        - Redis：内存数据库，暂存对话context
        - MinIO/AWS S3：存储用户的音频
    - AI与核心算法：
        - llm：对话生成引擎，需设计system prompt
        - ASR：语音识别（语音转文字）
        - Azure Speech SDK/讯飞开放平台：发音质量评估
