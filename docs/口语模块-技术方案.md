## 2. 组件选型

| 组件 | 选型 | 理由 |
|------|------|------|
| **ASR** 语音→文字 | 讯飞语音听写 | 业界领先，免费额度够用 |
| **LLM** 对话生成 | Qwen (通义千问) / DeepSeek | 英文强，价格低，API兼容OpenAI |
| **TTS** 文字→语音 | 讯飞语音合成 | 免费每天500次 |
| **ISE** 发音评分 | 讯飞语音评测 | 专业发音评分，免费每天500次 |
| **前端** | React (已有) | 现有项目技术栈 |
| **后端** | Spring Boot 3 (已有) | 现有项目技术栈 |

## 3. 核心交互流程

### 3.1 一次对话轮次

```
用户点击录音
  │
  用户说话（英文）
  │
  停止录音
  │
  POST /api/speaking/sessions/{id}/messages  (multipart: 音频文件)
  │
  ┌── 后端并行处理 ──────────────────────────────────┐
  │                                                  │
  │  分支A: 语音识别                                  │
  │  音频 → 讯飞语音听写(ASR) → transcribed_text     │
  │                                                  │
  │  分支B: 发音评测                                  │
  │  音频 → 讯飞ISE → pronunciation_score(存库)      │
  │                                                  │
  │  分支A续: 大模型对话                              │
  │  transcribed_text + 对话历史 + system prompt     │
  │     → Qwen/DeepSeek → agent_reply_text           │
  │                                                  │
  │  分支A续: 语音合成                                │
  │  agent_reply_text → 讯飞TTS → agent_audio_url    │
  │                                                  │
  └──────────────────────────────────────────────────┘
  │
  返回前端:
  {
    userMessage: { text: transcribed_text },
    agentMessage: { text: agent_reply_text, audioUrl: agent_audio_url },
    pronunciationScore: { totalScore, accuracy, fluency, wordScores },
    session: { currentTurn, status, ... }
  }
  │
  前端展示:
  ├── 用户气泡: transcribed_text
  ├── Agent气泡: agent_reply_text + 自动播放音频
  └── (本轮发音评分暂不展示，留到反馈页)
```

### 3.2 会话结束

```
用户点击"交卷"
  │
  GET /api/speaking/sessions/{id}/feedback
  │
  后端返回:
  ├── 每轮对话记录 (文字 + 发音评分)
  ├── 综合发音报告 (平均分、各维度趋势)
  ├── Agent综合评价
  └── 改进建议
```

## 4. Agent 接入设计

### 4.1 现有接口结构（不改）

```java
// SpeakingAgentClient.java — 无需修改
public interface SpeakingAgentClient {
    SpeakingAgentReply reply(
        SpeakingScenario scenario,
        List<SpeakingMessage> history,
        String userMessage,
        int turnIndex
    );
}
```

### 4.2 实现类替换

`MockSpeakingAgentClient` → `RealSpeakingAgentClient`

主要职责：
1. 从 `SpeakingScenario` 构造 system prompt
2. 收集对话历史拼成 messages 数组
3. 调用 LLM API (OpenAI 兼容格式)
4. 解析返回结果

### 4.3 System Prompt 模板（完整版）

```
## 角色定义
你是一个英文口语练习 AI 助教。你的唯一任务是在指定的场景中与用户进行英文对话练习。
你不是聊天机器人，不是搜索引擎，不是翻译工具，不是作文批改工具。你只做一件事：场景对话。

## 当前场景
- 场景名称：{title}
- 角色设定：{rolePrompt}
- 用户级别：{difficulty}
- 练习目标：{goal}
- 关键词：{keywords}
- 评分标准：{scoringRubric}
- 总对话轮次：{targetTurns} 轮（包含开场白）
- 当前轮次：第 {currentTurn} 轮

## 输出格式（严格遵守）

每次回复必须遵循以下格式之一：

【格式A — 正常回复】（most cases）
只输出纯英文文本，不加任何标记、注释、前缀或后缀。
不要使用 Markdown 格式，不要加粗，不要用列表符号。
示例：
  Good morning. May I see your passport and booking reference?

【格式B — 结束轮次】（最后一轮）
只输出纯英文文本，末尾加 [END]。
示例：
  Your boarding pass is ready. Have a great flight! [END]

【格式C — 用户说中文时】
如果你收到中文消息，回复以下固定文本，不做任何变通：
  Please try saying that in English. I'm here to help you practice.

## 对话规则（必须遵守）

### 规则 1：回复长度
- 每次回复严格控制在 **1-2 句话**
- 不超过 40 个单词
- 不要一次说太多，给用户留说话空间

### 规则 2：语言
- 只用英文，一个中文字都不要出现
- 使用与用户级别（{difficulty}）匹配的词汇和句式
- 不要使用俚语、网络用语或生僻词

### 规则 3：角色扮演
- 始终保持在场景角色中，不要跳出
- 不要提及你是 AI、大模型、语言模型
- 不要解释你在做什么、为什么这么回复
- 不要说 "As an AI..." "I'm here to help..." 之类的话
- 不要主动提供额外的学习建议或词汇讲解

### 规则 4：语法纠错方式
- 如果用户有语法错误，**不要直接指出错误**
- 在回复中自然地示范正确说法即可
- 示例：
  用户说: "Yesterday I go to library."
  你的回复: "Oh, you went to the library yesterday? Which one?"
  （不要写: "You should say 'went' instead of 'go'."）

### 规则 5：话题边界
- 只围绕当前场景的主题进行对话
- 如果用户问与场景无关的问题，回复以下固定文本：
  Let's focus on the {场景名} practice for now.
- 不要讨论政治、宗教、医疗建议等敏感话题
- 不要询问用户的真实姓名、地址、电话等个人信息

### 规则 6：会话流程
- 第一轮：用 {openingMessage} 开场
- 中间轮次：根据用户的回答自然推进场景
- 最后一轮（第 {targetTurns} 轮）：自然结束对话，末尾加 [END]
- 不要提前结束，也不要超出设定的轮次

### 规则 7：角色边界
- 你是**对话练习伙伴**，不是考官
- 不要给用户的表现打分或评级
- 不要评价用户说得"好"或"不好"
- 不要说 "Good job!" "Well done!" "Great answer!" 之类的话
- 用自然的对话推进，而不是教学式的反馈

### 规则 8：禁止行为
- ❌ 不要使用 emoji
- ❌ 不要使用 Markdown 格式
- ❌ 不要加引号、星号、下划线
- ❌ 不要输出中文
- ❌ 不要解释或翻译英文
- ❌ 不要列出单词表
- ❌ 不要提供语法讲解
- ❌ 不要问用户想练习什么（场景已经定好了）

## 示例对话参考
{sampleDialogue}

记住：你的回复要简短、自然、在角色中。你是一个场景中的对话伙伴，仅此而已。
```

### 4.4 LLM 调用示例 (OpenAI 兼容格式)

```java
// 请求
POST https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
Authorization: Bearer sk-xxxxxxxx

{
  "model": "qwen-turbo",
  "messages": [
    { "role": "system", "content": "你是机场值机人员..." },
    { "role": "user", "content": "Hello, I'd like to check in." },
    { "role": "assistant", "content": "Of course. May I see your passport?" },
    { "role": "user", "content": "Here you are." }
  ],
  "temperature": 0.7,
  "max_tokens": 500
}

// 响应
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Thank you. Do you have any bags to check in today?"
      }
    }
  ]
}
```

## 5. 数据模型

### 5.1 现有表补充字段

**speaking_messages** 表新增字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `audio_url` | VARCHAR(500) | 用户录音文件地址 |
| `transcribed_text` | VARCHAR(4000) | ASR 转写结果 |
| `pronunciation_score` | DOUBLE | ISE 总分 |
| `pronunciation_detail` | JSON | ISE 详细评分（各维度+单词级）|

### 5.2 反馈结果 DTO

```java
record SpeakingFeedbackResponse(
    String scenarioTitle,
    int totalTurns,
    double averagePronunciationScore,
    List<TurnFeedback> turns,
    String agentOverallComment
)

record TurnFeedback(
    int turnIndex,
    String userText,
    String agentText,
    PronunciationScore score
)

record PronunciationScore(
    double totalScore,       // ISE 总分
    double accuracy,         // 准确度
    double fluency,          // 流利度
    double integrity,        // 完整度
    List<WordScore> words    // 单词级评分
)

record WordScore(
    String word,
    double score,
    int errorType            // 0正常 16漏读 32增读 64回读 128替换
)
```

## 6. 实现步骤

### 阶段一：LLM Agent 替换
- 注册 LLM API (Qwen/DeepSeek)
- 编写 `RealSpeakingAgentClient.java`
- 设计 system prompt 模板
- 替换 Mock，纯文字对话先跑通

### 阶段二：讯飞语音集成
- 注册讯飞开放平台，获取 AppID/APIKey/APISecret
- 编写 `IseService.java`（发音评测 WebSocket 客户端）
- 编写 `AsrService.java`（语音听写 WebSocket 客户端）
- 编写 `TtsService.java`（语音合成 WebSocket 客户端）

### 阶段三：前端录音改造
- 实现浏览器录音 (MediaRecorder → PCM 16k)
- 改造 `SpeakingConversationPage.jsx`
- POST 音频到后端

### 阶段四：全流程联调
- ASR + LLM + TTS 闭环
- 并行 ISE 评分
- 反馈页展示发音报告
